虚拟系统安装

对于生信学习来说，linux的学习是必不可少的，对于多数初学者来说，肯定是没有装好系统的集群来给大家使用，而且集群并不适合初学者学习和了解这个系统，因为学习的最好方式就是不断的尝试，所以一个大的集群可不能给大家拿来折腾，虚拟系统在功能上完全没用任何的区别（当然大的集群可能在提交任务等有那么一点区别）,我们可以拆开掰碎了去了解这个系统，系统搞坏了，大不了就是重新装一个。

1. 虚拟机软件及系统镜像准备

  • 虚拟机软件

    虚拟机就是运行虚拟系统的一个容器，同类软件也有好几种，我选择了VMware Workstation Pro，总体来说可以满足我的需求。

  • 系统镜像的准备

    • Ubuntu

      这个发行版的桌面比较友好，桌面版安装也比较人性化。最新版为17，当然16更加的成熟稳定。

      下载页面

    • bio-linux

      这个发行版实际上就有点相当于Windows系统预装了一些软件做成了镜像，同理，它就是Ubuntu预装了一些我们常用的生信软件，优点就是免去了很多我们装软件的烦恼，当然这不太利于我们对于linux软件编译安装的学习。

    • centos

      对于初学者来说，我还是比较推荐Ubuntu的，但是如果进行后期一点的学习，我还建议安装centos，因为我们的学习很多时候都是为了使用集群做准备的，而Ubuntu和centos在操作上还是有些区别的。

      ​

2. 虚拟系统的安装

下面我就以VMware Workstation Pro安装bio-linux为例演示虚拟系统的安装

  1. 系统的安装

    1. 安装VMware Workstation Pro软件，打开软件新建一个虚拟机。

      ​

    2. 选择自己要安装的镜像，这里就选择bio-linux。

    3. 之后就一直点下一步就好了。

    4. 第一次打开虚拟机，注意这个时候系统还并没有装好。开启虚拟机。

    5. 之后就后一般的装系统一样。一直选择下一步即可，分区就按默认的设置就行。

    6. 虚拟系统的硬件设置最好是不要超过你电脑硬件的一半，如果你不想你的电脑开个虚拟机就卡的不行的话。当然内存小于8G的电脑也不要装什么虚拟机了。

    7. 话说到此为止，你已经有一个可以使用的虚拟系统了。

  2. 如何舒服的使用你的虚拟系统。

    这个时候的虚拟系统并不完美，还存在以下两个问题。

    1. 屏幕分辨率的问题

      这个问题可以通过安装虚拟机的插件VMware Tool来解决。

      参考教程

    2. 虚拟系统和电脑交互的问题。

      1. 共享文件夹问题

        前提是已经装好了VMware Tool，在虚拟系统设置设置一个共享文件夹tmp，然后我们就可在虚拟系统的/mnt/htfs/目录下找到我们的共享文件夹了。我们可以把我们要操作的比较大一点的文件放在这个文件夹里，这个过程实质就是一个盘的挂载。

      2. ssh服务的问题

        其实大多数的发行版应该都是装了这个服务的，当然为了确保成功，不妨用了apt-get install ssh再装一遍。这个时候我们就可以ifconfig命令获得虚拟系统的ip地址了。这个时候我们就可以用Xshell等终端模拟器愉快的访问我们的虚拟系统了。

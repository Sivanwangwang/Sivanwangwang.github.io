虚拟系统安装

对于生信学习来说，linux的学习是必不可少的，对于多数初学者来说，肯定是没有装好系统的集群来给大家使用，而且集群并不适合初学者学习和了解这个系统，因为学习的最好方式就是不断的尝试，所以一个大的集群可不能给大家拿来折腾，虚拟系统在功能上完全没用任何的区别（当然大的集群可能在提交任务等有那么一点区别）,我们可以拆开掰碎了去了解这个系统，系统搞坏了，大不了就是重新装一个。

1. 虚拟机软件及系统镜像准备

  • 虚拟机软件

    虚拟机就是运行虚拟系统的一个容器，同类软件也有好几种，我选择了VMware Workstation Pro，总体来说可以满足我的需求。

  • 系统镜像的准备

    • Ubuntu

      这个发行版的桌面比较友好，桌面版安装也比较人性化。最新版为17，当然16更加的成熟稳定。

      下载页面

    • bio-linux

      这个发行版实际上就有点相当于Windows系统预装了一些软件做成了镜像，同理，它就是Ubuntu预装了一些我们常用的生信软件，优点就是免去了很多我们装软件的烦恼，当然这不太利于我们对于linux软件编译安装的学习。

    • centos

      对于初学者来说，我还是比较推荐Ubuntu的，但是如果进行后期一点的学习，我还建议安装centos，因为我们的学习很多时候都是为了使用集群做准备的，而Ubuntu和centos在操作上还是有些区别的。

      ​

2. 虚拟系统的安装

下面我就以VMware Workstation Pro安装bio-linux为例演示虚拟系统的安装

  1. 系统的安装

    1. 安装VMware Workstation Pro软件，打开软件新建一个虚拟机。

      ​

    2. 选择自己要安装的镜像，这里就选择bio-linux。

    3. 之后就一直点下一步就好了。

    4. 第一次打开虚拟机，注意这个时候系统还并没有装好。开启虚拟机。

    5. 之后就后一般的装系统一样。一直选择下一步即可，分区就按默认的设置就行。

    6. 虚拟系统的硬件设置最好是不要超过你电脑硬件的一半，如果你不想你的电脑开个虚拟机就卡的不行的话。当然内存小于8G的电脑也不要装什么虚拟机了。

    7. 话说到此为止，你已经有一个可以使用的虚拟系统了。

  2. 如何舒服的使用你的虚拟系统。

    这个时候的虚拟系统并不完美，还存在以下两个问题。

    1. 屏幕分辨率的问题

      这个问题可以通过安装虚拟机的插件VMware Tool来解决。

      参考教程

    2. 虚拟系统和电脑交互的问题。

      1. 共享文件夹问题

        前提是已经装好了VMware Tool，在虚拟系统设置设置一个共享文件夹tmp，然后我们就可在虚拟系统的/mnt/htfs/目录下找到我们的共享文件夹了。我们可以把我们要操作的比较大一点的文件放在这个文件夹里，这个过程实质就是一个盘的挂载。

      2. ssh服务的问题

        其实大多数的发行版应该都是装了这个服务的，当然为了确保成功，不妨用了apt-get install ssh再装一遍。这个时候我们就可以ifconfig命令获得虚拟系统的ip地址了。这个时候我们就可以用Xshell等终端模拟器愉快的访问我们的虚拟系统了。

biotar 学习笔记（2-2）--- 环境配置、数据的重复性

发表于 2017-11-23 | 分类于 biostar

系统安装完毕了，就存在进行软件的安装。

对于linux来说，主要包括三种软件安装方式：

  1. 绿色软件，解压可用。
  2. 源代码的文件，需要编译。
  3. 使用系统软件商店或者叫镜像也行。

当然前两种方式对于大多数人来说，都是比较麻烦的，有没有像第三种方式一样能把大部分软件通过一句命令就能安装呢？答案是有的，那就conda。这个的总结洲更班长总结的已经很完善。

洲更conda笔记三部曲

2.生信软件的好帮手-bioconda
3.什么，你嫌bioconda下载速度太慢？

配置可重复性。

这个是最近才有的体会，比如说你有好几个电脑都想有相同分析环境，这个不免会把有的软件可能会重新装一遍.

  1. 那么这个时候，最好是不要去乱去重新一个新的文件夹。比如说在电脑1上使用biosoft文件夹来装软件，那么在电脑2上也请用biosoft文件夹，因为的记录一次的安装代码可以随意的重复使用。
  2. 请善待路径中的~，你会喜欢上它的。
  3. 如果是联网的电脑，一个软件安装脚本就可以帮你搞定一台新的电脑的配置。
  4. 当然环境的移植，conda也可以，或者还有其他的方式，有待发掘。

数据的重复性

我的理解有以下几点：

  1. 生信的学习必经之路，这个就跟我们做完题，会去对一对标准答案一样的。
  2. 数据的重复，必须基于的是软件及其版本一致，基因版本，注释文件版本和数据库版本一致。
  3. 测序的数据不应该一次用完就抛在一边，那样太可惜了，因为数据库在更新，以及联合分析多篇文章的数据都可能有新的科学发现。

biostar 学习笔记1---为什么要学习生信

发表于 2017-11-23 | 分类于 biostar

不跟进先进生产力，你将被时代所淘汰。

这不禁让我想到90年代，倒闭了一大批的企业，当然这里边不乏很多其他的问题，但是有个重要的问题，就是技术落后。可能当时很多人就很疑惑，我们很努力，我们也勤奋，但是为啥就被淘汰了，因为你没有跟上先进的生产力。高通量测序技术这种新事物的产生，和一切其他的新事物一样，有的人先去吃了螃蟹，有的人处于观望的态度，有的人不以为然。大数据是一个时代，并不局限于生物领域，而且它正在不断的证明自己是一种先进生产力。生物学科发展那么多年，早已经进入了分子时代，以前由于技术限制，只能单个或者少数几个基因，少量的生物个体进行研究。而这种东西在大数据面前这种研究模式不能继续成为理所当然，因为我们不能一直管中窥豹，因为想看得更远，只能站的更高，一定要有大局观。人有两万左右的编码基因，加上一些其他基因有4万多，如果还是现在的基本分子生物学将无法进行研究，即使耗费足够的精力，也无法研究清楚，因为机体是个整体，牵一发而动全身，不通过组学将很难全局的观测到发生的变化。

学习伴随人的一生。

学习总是一个被动和主动的过程，或者说交替向前。生信专业在近几年在各大院校已经变得较为普遍了，这些专业的学生很大一部分属于自己选择专业，兴趣学习，学习也比较循序渐进。但是很多时候，学习的安排并不是那么理想化，现在很多学习生信的人很多都是半路出家，有学生、老师、医生、公司从业人员等等，这一类人都是属于按需学习。当然我也属于这一类人。我的专业是动物遗传学，研究物种是猪。貌似好像我本来就该学点生信，其实并没有，我已经做了四年的实验，接触生信也才一年，都是被逼的，因为猪这个物种本来就基础研究薄弱，好多方面没有别人测的数据进行借鉴，或者说别人的文章的结果没有自己关心的数据，有或者说自己想结合自己的数据和别人数据，再或者说比较一下不同物种的数据，反正总之就是公司肯定是没有这么多的自定义套餐的，你不能遇到个这种问题就外包公司吧？都是自己会比较得心应手。对一开始学习生信，我特别的痛苦，因为不知道往哪里走，但是回过头一想想，其实研究生的学习不就是锻炼发现问题，解决问题的能力。学习就是解决问题的一个手段而已。一开始找各种各样的资料，成为资料收集爱好者，哈哈！找相关的QQ群、论坛、公众号。其实还是挺不错的，起码发现还有比自己菜的人，开个玩笑。一起相互监督学习不乏问一种好方法，其实不管学习什么，有收获总是你继续下去的动力，比如有一个天你终于码出一个不在重复简单动作的脚本，那种开心真的就是单纯的开心。所以不管学习的开端是痛苦的，还是快乐的，始终支撑你学习下去的还是兴趣和快乐，当然有的解决完问题可能就先放一边了，不过现在的我，可能即使目前不需要能力，我也想去学习，因为学习生信本身就让我很快乐。

永远不应该忽视的生物学问题

既然叫做生物信息，当然就不应该是个真正的码农，因为我们的一切目的还是得回归解决生物学问题。有时自己也会陷入这样的一个陷阱，就是觉得别人的图好漂亮，我要学会，当然学会了也会很有成就感,其实这并没有有助我们生物学问题的解决。我们不同于公司的人，也不同于真正的程序员，因为我们有更好的生物学基础，所以才能更好的利用生物信息学解决我们自己的问题。

写markdown要具备的软件

本地软件安装

1. Markdownpad

http://markdownpad.com/

需要环境

http://markdownpad.com/download/awesomium_v1.6.6_sdk_win.exe

优缺点

网络上说了一大堆的有点，我觉得基本属于没优点，缺点倒是一大堆，对于代码的语法不能用”```“,还有就是免费版不能导出pdf,不能同步到GitHub及其他云端。

2.dillinger

dillinger是一个在线软件，具备几乎所有的优点，对于代码的兼容性特别好，并且还可以免费保存成HTML、Styled HTML、Markdown、PDF的格式，还可以同步到Dropbox、GitHub、Medium、Google Drive、One Drive。唯一的不足就是在线软件，所有也不存在云储存空间，也就无法通过直接粘贴来建立图片链接。

http://dillinger.io/

3.leanote

leanote是一个在线和桌面客服端都有的软件。除了dillinger的大多数优点（保存的格式有PDF、HTML、Evernote、leanote,并且pdf格式30天免费期之后，会带有水印），并且不能同步到那么多的云端，但是有个特别好的功能就是通过粘贴来建立图片链接，并把图片保存到leanote的云空间。并且这个云笔记界面简洁，可以同步markdown文件。升级空间也巨大。而保存多样式和其他云端同步可以用dillinger来进行弥补。

4.有道云笔记

有道云笔记也存在代码兼容性差的问题

5.简书

简书虽然可以存私密的文章，但是浏览时关感太差，或者说基本无观感。而观感比较好的又必须发布。

总结
